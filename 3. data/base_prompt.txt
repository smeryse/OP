---

JSON должен иметь следующую структуру:

```
{
  "lab": {
    "number": <номер>,
    "discipline": "Операционные системы",
    "theme": "<название лабораторной работы>"
  },
  "goals": "<цель работы>",
  "procedure": [
    {
      "text": "<текст шага>",
      "images": [
        {
          "number": <N>,
          "src": "images/<N>.png",
          "caption": "<описание>"
        }
      ]
    }
  ],
  "questions": [
    {
      "question": "<вопрос>",
      "answer": "<ответ>"
    }
  ],
  "conclusion": "<вывод>"
}
```

Требования к заполнению полей:

lab.theme
Просто название лабораторной работы без лишнего текста.
Пример: "Изучение работы с консольным языком Bash".

goals
Строка формата:
"Цель работы: <цель из лабораторного файла>".
Записывается одной строкой.

procedure
Соответствует блокам "Ход работы" и "Практическое задание". Теоретические части без действий не включаются.

Каждый элемент массива procedure — один выполненный шаг.

Требования к text:

* описывает только выполненные действия;
* строгий формальный стиль;
* 1–2 предложения по существу;
* небольшие действия можно объединять, но существенные действия описывать отдельно;
* нельзя объединять слишком много действий в один шаг;
* в конце предложения обязательно должна быть ссылка на рисунок в формате "как показано на рисунке N";
* текст должен соответствовать описанию изображения.

Требования к images:

* содержит изображение, относящееся к шагу;
* number — номер рисунка;
* src — "images/N.png";
* caption — описание изображения строго в формате: "Рисунок N — описание";
* используется длинное тире (—);
* нумерация рисунков сквозная по всей работе;
* нумерация не начинается заново в практическом задании.

questions
Соответствует блоку "Вопросы".

Требования:

* каждый объект содержит вопрос и развернутый ответ;
* ответы должны быть подробными и формальными;
* вопрос записывается полностью;
* ответ даётся отдельно в поле answer.

conclusion
Соответствует блоку "Вывод".

Требования:

* краткое формальное описание выполненной работы;
* что было изучено, выполнено и получено в результате лабораторной.

Общие требования:

* стиль строгий и формальный;
* описываются только выполненные действия;
* никаких теоретических рассуждений без действий;
* структура JSON должна строго соблюдаться;
* текст должен быть без лишнего форматирования.
